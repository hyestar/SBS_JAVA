**클래스**

1. class라는 것은 틀, 설계도 역할을 한다.
2. 객체화(class의 복사본 만드는 것)
3. 만들어진 복사본을 객체, 인스턴스, 오브젝트라고 부른다.
4. 만들어진 객체는 이름이 없다. 구분하기 위해서 변수와 연결한다.
5. 객체의 변수는 타입이 class명과 같아야 한다.
